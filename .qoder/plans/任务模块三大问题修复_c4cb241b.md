
# 任务模块三大问题修复

## Task 1: 修复任务配置保存 — toRaw 序列化

**文件**: `src/renderer/src/pages/tasks.vue`
- `saveTask()` 函数中 `taskSettings.value` 未做 `toRaw()` 处理，Vue 响应式 Proxy 通过 IPC 序列化时可能丢失数据
- 修复：使用 `JSON.parse(JSON.stringify(toRaw(taskSettings.value)))` 确保干净的对象传递

## Task 2: 修复 CRUD 任务 ID 与运行时 ID 的关联（暂停/恢复/停止的核心问题）

**根因**: CRUD 任务 ID (`task-xxx`) 和 TaskRunner 运行时 ID (`tsk-xxx`) 是两套独立 ID 体系，UI 用 CRUD ID 去 `runningTasks` 里查找，永远匹配不上，导致暂停/恢复按钮不显示、停止操作变成停止全部

**修改文件**:
1. `src/main/service/task-runner.ts` — TaskRunConfig 新增 `crudTaskId` 字段，TaskRunner 存储 `crudTaskId`
2. `src/main/service/task-manager.ts` — TaskStatusInfo 新增 `crudTaskId` 字段，`getAllRunningTasks()` / `getTaskStatus()` 返回 crudTaskId；`startTask()` 传递 crudTaskId
3. `src/main/ipc/task.ts` — `task:start` 接收并传递 `crudTaskId`
4. `src/preload/index.ts` — `task.start` config 类型新增 `crudTaskId`
5. `src/renderer/src/stores/task.ts` — `start()` 传递 `crudTaskId`；`RunningTaskInfo` 新增 `crudTaskId` 字段；`runningTasks` 改为用 `crudTaskId` 做 Map 索引
6. `src/renderer/src/pages/tasks.vue` — `currentRunningTaskInfo` 改用 `crudTaskId` 匹配；`stopTask()` / `pauseCurrentTask()` / `resumeCurrentTask()` 传递正确的 runner taskId

## Task 3: 修复抖音视频信息获取

**修改文件**:
1. `src/main/service/task-runner.ts` — 
   - `setupVideoDataListener()`: 增加 URL 匹配模式（匹配 `/aweme/v1/web/` 路径段，而非完整 URL），增加详细日志
   - `getCurrentVideoInfo()`: 增加重试等待时间（500→1000ms），增加日志输出每一轮获取状态
   - 新增 `getVideoInfoFromDOM()` 降级方法：当 videoCache 未命中时，从页面 DOM 直接提取视频信息（描述、作者等）
   - `startWithContext()` / `start()`: 页面加载后增加等待，确保 feed 数据先到达

2. `src/main/platform/douyin/index.ts` — 
   - `getVideoInfo()`: 当 videoCache 未命中时，尝试从页面 DOM 元素提取基础信息作为降级
   - 新增 `getVideoInfoFromDOM()` 方法：从 `data-e2e` 属性的 DOM 元素提取视频描述、作者昵称等
