# 任务启动无反应问题排查计划

## 问题描述
点击任务界面的"启动"按钮没有任何反应，既不报错也不弹出提示。

## 可能原因分析

### 1. IPC 通道未正确连接
前端 `runtimeTaskStore.start()` 调用 `window.api.task.start()`，但主进程可能未正确响应。

### 2. TaskRunner 初始化或执行异常
`TaskRunner.start()` 在执行过程中可能抛出异常但未被捕获。

### 3. 平台适配器问题
`DouyinPlatformAdapter.setVideoCache()` 方法存在但未被正确使用（被派生类同名私有字段遮蔽）。

## 排查步骤

### 步骤 1: 添加前端调试日志
在 `src/renderer/src/stores/task.ts` 的 `start()` 函数中添加 console.log 确认调用是否到达。

### 步骤 2: 检查主进程日志
查看 Electron 主进程的日志输出，确认 `task:start` IPC handler 是否被调用。

### 步骤 3: 检查 Browser path 配置
`task.ts` 中如果 `browserExecPath` 未配置，会返回错误但可能前端未显示。

### 步骤 4: 修复 DouyinPlatformAdapter.setVideoCache 问题
`DouyinPlatformAdapter` 有私有 `videoCache` 字段遮蔽了 `BasePlatformAdapter` 的 `setVideoCache()` 方法。

## 修复方案

### 修复 1: 在 task.ts IPC handler 中添加更多错误处理和日志
```typescript
// 在 task.ts 中
log.info('Received task:start request', config)
const browserExecPath = store.get(StorageKey.BROWSER_EXEC_PATH)
if (!browserExecPath) {
  log.error('Browser path not configured')
  return { success: false, error: 'Browser path not configured' }
}
```

### 修复 2: 确保 onAction 事件正确注册
preload 中使用 `ipcRenderer.on('task:action', ...)` 但主进程发送的是 `'task:action'`。

### 修复 3: 修复 DouyinPlatformAdapter 的 videoCache 遮蔽问题
将 `DouyinPlatformAdapter` 中的 `private videoCache` 改为使用继承的方法，或重写 `setVideoCache` 方法。

## 实施步骤

1. 在 `src/main/ipc/task.ts` 中添加详细日志
2. 在 `src/renderer/src/stores/task.ts` 中添加 try-catch 和日志
3. 修复 `DouyinPlatformAdapter` 的 videoCache 问题
4. 测试验证任务启动功能
