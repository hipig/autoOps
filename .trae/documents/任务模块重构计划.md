# 任务模块重构计划

## 一、现状分析

### 1.1 当前架构

```
src/shared/
├── task.ts           # Task, TaskTemplate 接口定义
└── platform.ts       # Platform, TaskType 定义 (comment|lake|collect|follow|watch|combo)

src/main/
├── ipc/
│   ├── task.ts       # 任务启动/停止 (start/stop/status)
│   └── task-crud.ts  # 任务CRUD操作
├── service/
│   └── douyin-task.ts # 抖音任务实现 (当前仅支持评论)
├── platform/
│   ├── base.ts       # BasePlatformAdapter 抽象类
│   ├── factory.ts    # 平台适配器工厂
│   └── douyin/index.ts
└── elements/
    └── douyin.ts     # 抖音元素选择器

src/renderer/src/
├── pages/tasks.vue   # 任务管理UI
└── stores/
    ├── task.ts       # 运行时任务状态 (启动/停止/日志)
    └── task-new.ts   # 任务CRUD状态管理
```

### 1.2 存在的问题

#### 问题1: 任务类型单一，仅评论功能可用

**代码证据** (`src/main/service/douyin-task.ts`):
- `runTask()` 方法中只有 `executeComment()` 调用
- `like()` 方法存在于 `DouyinElementSelector` 但从未被正式任务调用
- `collect()`, `follow()` 等操作完全没有实现

**TaskType 定义** (`src/shared/platform.ts`):
```typescript
export type TaskType = 'comment' | 'like' | 'collect' | 'follow' | 'watch' | 'combo'
```
定义了6种类型，但实际只有 `comment` 实现。

#### 问题2: 任务启动功能异常

**代码证据** (`src/main/ipc/task.ts`):
```typescript
currentTask = new DouyinTask()  // 硬编码创建 DouyinTask
```

**问题**:
- 虽然存在 `src/main/platform/factory.ts` 工厂模式，但 `task.ts` 完全未使用
- 任务启动时没有根据平台类型选择对应的适配器
- 没有根据 `taskType` 选择执行不同的操作

#### 问题3: 架构设计问题

- `DouyinTask` 直接继承了 `EventEmitter`，而非 `BasePlatformAdapter`
- `BasePlatformAdapter` 定义了抽象方法 `like()`, `collect()`, `follow()`, `comment()`，但未被使用
- 配置 `FeedAcSettingsV2` 是针对评论设计的，不适合其他任务类型

## 二、重构目标

### 2.1 功能目标

1. **支持多种任务类型**: 评论、点赞、收藏、关注、观看、组合任务
2. **修复任务启动功能**: 使用工厂模式正确创建平台适配器
3. **统一的任务执行流程**: 适配器模式 + 策略模式
4. **可配置的任务组合**: 支持在同一任务中执行多种操作

### 2.2 架构目标

                    ┌──────────────▼──────────────────────┐

         ┌─────────────────────────┼─────────────────────────┐

#### 步骤1.1: 扩展 TaskType 定义
**文件**: `src/shared/platform.ts`

```typescript
export enum TaskOperation {
  COMMENT = 'comment',
  LIKE = 'like',
  COLLECT = 'collect',
  FOLLOW = 'follow',
  WATCH = 'watch'
}

export type TaskType = TaskOperation | 'combo'
```

#### 步骤1.2: 创建任务操作配置
**文件**: `src/shared/task-operation.ts` (新建)

```typescript
export interface TaskOperationConfig {
  type: TaskOperation
  enabled: boolean
  probability: number  // 执行概率 0-1
  maxCount?: number   // 最大执行次数
  aiEnabled?: boolean // 是否使用AI生成
}

export interface ComboTaskConfig {
  operations: TaskOperationConfig[]
  stopOnFirstSuccess?: boolean  // 组合任务中某个成功后是否停止
}
```

#### 步骤1.3: 更新 FeedAcSettingsV2
**文件**: `src/shared/feed-ac-setting.ts`

为每种操作类型添加独立的配置支持。

## 六、风险和注意事项

1. **向后兼容**: 确保现有任务数据能正常加载
2. **错误处理**: 每种操作都需要完善的错误处理
3. **验证码处理**: 点赞、收藏等操作可能触发验证码
4. **频率限制**: 不同操作有不同的频率限制，需要添加延迟
5. **随机性**: 引入概率控制，避免被检测为机器人
